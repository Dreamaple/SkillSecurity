# SkillSecurity 架构概述

> **版本**: 0.2.0  
> **日期**: 2026-03-11  
> **关联**: [主需求文档](../SkillSecurity需求规格说明书.md) 第 4 节

---

## 1. 设计原则

| 原则 | 说明 |
|---|---|
| **零侵入** | 不要求修改 Skill 代码，通过包装层或代理实现拦截 |
| **默认安全** | 开箱即用的默认策略，无需配置即可防护常见风险 |
| **可插拔** | 每个组件可独立启用/禁用/替换 |
| **本地优先** | 默认纯本地运行，不依赖外部服务 |
| **最小依赖** | 核心引擎尽量减少第三方依赖（降低自身攻击面） |
| **性能优先** | 拦截延迟 < 10ms（平均），不拖慢 Agent 运行 |

### 2.2 核心组件

| 组件 | 职责 | 阶段 |
|---|---|---|
| **Interceptor（拦截器）** | 钩入工具调用链，捕获调用请求 | Phase 1 |
| **Policy Engine（策略引擎）** | 加载策略配置，匹配规则 | Phase 1 |
| **Decision Maker（决策器）** | 根据匹配结果做出 Allow/Block/Ask 决策 | Phase 1 |
| **Static Analyzer（静态分析器）** | 扫描 Skill 代码中的危险模式 | Phase 2 |
| **Permission Manager（权限管理器）** | 管理 Skill 权限声明与运行时权限边界 | Phase 2 |
| **Audit Logger（审计日志）** | 记录所有操作和决策 | Phase 2 |
| **Alert Notifier（告警通知）** | 向配置的渠道发送告警 | Phase 4 |
| **User Prompter（用户交互）** | Ask 决策时请求用户确认 | Phase 4 |

## 3. 集成模式

SkillSecurity 提供两种集成模式，满足不同场景需求：

### 3.1 嵌入模式（Embedded Mode）

SDK 直接嵌入 Agent 进程，零网络开销。

```
┌─────────────────────────────────┐
│          Agent Process          │
│                                 │
│  Tool Call ──▶ [SDK] ──▶ Tool  │
│                 │               │
│              (进程内)            │
└─────────────────────────────────┘
```

**适用场景**：个人开发者、单 Agent 部署、对延迟敏感

**优点**：
- 零网络延迟
- 部署简单（安装 SDK 即可）
- 无额外进程管理

**缺点**：
- 策略变更需重启 Agent（或支持热加载）
- 日志在 Agent 进程内，需额外配置持久化
- 每个 Agent 独立，无集中管理

### 3.2 服务模式（Service Mode）

SDK 作为轻量客户端，将请求转发给独立运行的 SkillSecurity Server。

```
┌──────────────┐          ┌──────────────────────┐
│ Agent 1      │          │  SkillSecurity       │
│  [SDK Client]│──gRPC──▶│  Server              │
│              │          │                      │
├──────────────┤          │  - 策略引擎           │
│ Agent 2      │          │  - 审计日志           │
│  [SDK Client]│──gRPC──▶│  - 告警通知           │
│              │          │  - Web 管理界面       │
├──────────────┤          │                      │
│ Agent N      │          │                      │
│  [SDK Client]│──gRPC──▶│                      │
└──────────────┘          └──────────────────────┘
```

**适用场景**：团队/企业、多 Agent 部署、需要集中管理

**优点**：
- 集中策略管理
- 统一审计日志
- 支持 Web 管理界面
- 策略热更新不影响 Agent

**缺点**：
- 额外的网络延迟（同机 gRPC 通常 < 1ms）
- 需要部署和管理额外服务
- Server 成为单点（需考虑高可用）

### 3.3 模式选择指南

| 场景 | 推荐模式 | 理由 |
|---|---|---|
| 个人开发，单 Agent | 嵌入模式 | 最简单 |
| 多 Agent，想统一管理 | 服务模式 | 集中策略 |
| 生产环境，需要审计 | 服务模式 | 统一日志 |
| 对延迟极其敏感 | 嵌入模式 | 零网络开销 |
| 先试用再评估 | 嵌入模式 → 服务模式 | 渐进式 |

## 6. 技术选型

### 6.1 核心引擎

| 候选 | 优势 | 劣势 | 推荐度 |
|---|---|---|---|
| **Go** | 单二进制、交叉编译容易、并发模型好、生态丰富 | AI 社区不常用 | ★★★★☆ |
| **Rust** | 极致性能、内存安全、单二进制 | 学习曲线陡、编译慢 | ★★★★☆ |
| **Python** | AI 生态原生、开发快 | 性能瓶颈、分发复杂 | ★★★☆☆ |

**建议**：优先考虑 **Go**。理由：
- 编译为单二进制，`docker` / 直接下载即可部署
- 交叉编译简单，轻松覆盖 Linux / macOS / Windows
- goroutine 原生支持高并发拦截
- 有成熟的 gRPC 和 YAML 生态
- 比 Rust 开发效率更高，比 Python 性能更好

### 6.2 SDK

| 语言 | 形式 | 安装方式 | 优先级 |
|---|---|---|---|
| Python | `pip install skillsecurity` | PyPI | Phase 3 优先 |
| Node.js | `npm install skillsecurity` | npm | Phase 3 |
| Go | `go get` | Go Modules | Phase 3（如核心用 Go） |
| REST API | HTTP 调用 | 无需安装 | Phase 3（语言无关） |

### 6.3 存储

| 用途 | 方案 | 说明 |
|---|---|---|
| 策略配置 | YAML 文件 | 人类可读、版本控制友好 |
| 审计日志 | JSONL 文件（默认） | 无需数据库，`jq`/`grep` 可查 |
| 审计日志（大规模） | SQLite / PostgreSQL（可选） | 支持复杂查询 |
| 规则缓存 | 内存 | 启动时加载，热更新 |

### 6.4 通信协议

| 场景 | 协议 | 理由 |
|---|---|---|
| SDK ↔ Server（内部） | gRPC | 高性能、强类型、流式支持 |
| 外部 API（管理/查询） | REST + JSON | 通用性强、调试方便 |
| Webhook 告警 | HTTP POST | 标准方式 |

## 7. 部署架构

### 7.1 个人开发者（最小部署）

```
一条命令：pip install skillsecurity

Agent 代码中：
  from skillsecurity import guard
  result = guard.check(tool_call)
```

无需 Docker，无需独立服务，嵌入模式直接使用。

### 7.2 团队部署（Docker Compose）

```yaml
# docker-compose.yml 示例
services:
  skillsecurity:
    image: skillsecurity/server:latest
    ports:
      - "9090:9090"   # gRPC
      - "8080:8080"   # REST API / Web UI
    volumes:
      - ./policies:/etc/skillsecurity/policies
      - ./logs:/var/log/skillsecurity
```

### 7.3 企业部署（Kubernetes）

## 8. 可扩展性设计

### 8.1 插件机制

SkillSecurity 的关键扩展点均通过插件接口实现：

```
┌──────────────────────────────────────┐
│           Plugin Interface            │
├──────────────────────────────────────┤
│  Rule Plugin     │ 自定义拦截规则     │
│  Scanner Plugin  │ 自定义静态扫描规则  │
│  Notifier Plugin │ 自定义通知渠道     │
│  Storage Plugin  │ 自定义日志存储后端  │
│  Auth Plugin     │ 自定义认证方式     │
└──────────────────────────────────────┘
```

### 8.2 扩展示例

```go
// 自定义规则插件接口（概念示意）
type RulePlugin interface {
    Name() string
    Check(ctx context.Context, call ToolCall) Decision
}

// 自定义通知渠道插件接口
type NotifierPlugin interface {
    Name() string
    Send(ctx context.Context, alert Alert) error
}
```

## 9. 安全自保

SkillSecurity 自身也需要安全：

| 防护点 | 措施 |
|---|---|
| 策略文件 | 文件系统权限限制 + 可选签名验证 |
| API 端点 | API Key / JWT 认证 |
| 日志 | 敏感信息自动脱敏 |
| 依赖 | 定期 `govulncheck` / `npm audit` |
| 自身操作 | SkillSecurity 的配置变更也记录审计日志 |
| 降级策略 | 自身故障时可配置 fail-open 或 fail-close |

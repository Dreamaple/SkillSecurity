# Pro / Enterprise 扩展层

> **定位**: 开源核心之上的增值功能  
> **时间**: v1.0.0 之后持续迭代  
> **关联**: [主需求文档](../SkillSecurity需求规格说明书.md) 第 5 节

---

## 1. 层级划分

```
┌─────────────────────────────────────────────┐
│          Enterprise（商业版）                 │
│  SSO / GDPR / 异常检测 / 沙箱 / 合规报告     │
├─────────────────────────────────────────────┤
│          Pro（可选扩展）                      │
│  Web 管理界面 / 性能优化 / 高级查询           │
├─────────────────────────────────────────────┤
│          Core（开源核心）                     │
│  拦截 / 策略 / 权限 / 扫描 / SDK / 告警      │
│  Phase 1-4 全部内容                          │
└─────────────────────────────────────────────┘
```

**开源策略**：Core 层永久开源（Apache 2.0）。Pro 层可开源也可作为付费 SaaS 提供。Enterprise 层视商业模式决定。

## 2. Pro 层功能

### F10: Web 管理界面

#### 2.1 概述

提供可视化的管理和监控界面，替代纯 CLI 操作。

#### 2.2 功能页面

| 页面 | 功能 |
|---|---|
| **仪表盘** | 实时拦截监控、关键指标图表、今日摘要 |
| **事件流** | 实时滚动的拦截事件流，可过滤 |
| **日志查询** | 图形化的日志搜索、过滤、分页 |
| **策略管理** | 在线编辑策略、语法校验、版本历史 |
| **Skill 管理** | 已注册 Skill 列表、权限查看、扫描报告 |
| **告警管理** | 告警渠道配置、告警历史、静默规则 |
| **设置** | 系统配置、用户管理、API Key 管理 |

### F13: 性能优化

#### 2.6 概述

面向高负载场景的性能优化，确保在大规模部署下不成为瓶颈。

#### 2.7 优化目标

| 指标 | Core | Pro 优化后 |
|---|---|---|
| 平均延迟 | < 10ms | < 5ms |
| P99 延迟 | < 50ms | < 20ms |
| 吞吐量 | ≥ 1,000/秒 | ≥ 10,000/秒 |
| 内存 | < 50MB | < 100MB（支持更多缓存） |

#### 2.8 优化手段

| 优化 | 描述 |
|---|---|
| **策略预编译** | 启动时将正则编译为 DFA，匹配时间 O(n) |
| **Bloom Filter 预过滤** | 用布隆过滤器快速排除不匹配的请求 |
| **分层缓存** | 热门请求结果缓存，相同调用直接返回 |
| **并发处理** | 多条规则并行匹配 |
| **连接池** | gRPC 连接复用，减少建连开销 |
| **零分配路径** | 对于 Allow 请求最小化内存分配 |

#### 2.9 验收标准

- [ ] 有完整的 benchmark 报告
- [ ] 支持通过 Prometheus 暴露性能指标
- [ ] 提供性能调优指南
- [ ] 高负载下优雅降级（不崩溃、不无限队列）

## 3. Enterprise 层功能

### F11: 异常检测

#### 3.1 概述

基于行为模式自动检测异常，无需手动编写规则即可发现可疑行为。

#### 3.2 检测维度

| 维度 | 检测内容 | 示例 |
|---|---|---|
| **频率异常** | 短时间内大量操作 | 1 分钟内 50 次 shell 调用 |
| **时间异常** | 非常规时间的操作 | 凌晨 3 点执行大量写操作 |
| **序列异常** | 不寻常的操作组合 | 读取 .env → 网络请求（可能数据外泄） |
| **权限异常** | 频繁越权尝试 | 同一 Skill 反复尝试未声明的权限 |
| **偏差异常** | 偏离历史行为模式 | 平时只读文件的 Agent 突然大量写入 |

#### 3.3 检测方法

| 方法 | 适用场景 | 复杂度 |
|---|---|---|
| 统计阈值 | 频率异常、时间异常 | 低 |
| 滑动窗口 | 短时间内的行为聚合 | 中 |
| 序列分析 | 操作组合模式 | 中 |
| 基线比对 | 偏差异常 | 高 |

#### 3.4 响应机制

检测到异常后的自动响应：

```yaml
anomaly_detection:
  enabled: true
  response:
    - severity: high
      actions:
        - escalate_security_level: strict   # 自动提升安全级别
        - alert: [slack-security, webhook-siem]
        - log: detailed
    - severity: critical
      actions:
        - block_agent: true                 # 暂停该 Agent 的所有操作
        - alert: [all]
        - log: detailed
```

#### 3.5 验收标准

- [ ] 误报率 < 5%
- [ ] 检测延迟 < 1 分钟（近实时）
- [ ] 支持自定义检测规则
- [ ] 异常事件有详细上下文（不只是"发现异常"）

### F17: 沙箱预执行

#### 3.6 概述

在隔离环境中预先执行 Skill，观察其行为模式，作为安装前审查的补充手段。

#### 3.7 沙箱能力

| 能力 | 描述 |
|---|---|
| 文件系统隔离 | 使用临时目录，不影响主机文件系统 |
| 网络隔离 | 拦截或模拟网络请求 |
| 进程隔离 | 容器或 seccomp 沙箱 |
| 资源限制 | CPU、内存、执行时间上限 |
| 行为记录 | 记录所有系统调用和操作 |

#### 3.8 沙箱流程

```
Skill 代码
    │
    ▼
┌──────────────┐
│  创建沙箱     │  Docker 容器 / seccomp 进程
└──────┬───────┘
    │
    ▼
┌──────────────┐
│  执行 Skill   │  提供模拟输入
└──────┬───────┘
    │
    ▼
┌──────────────┐
│  行为记录     │  系统调用、文件访问、网络请求
└──────┬───────┘
    │
    ▼
┌──────────────┐
│  行为分析     │  与声明权限比对，检测异常行为
└──────┬───────┘
    │
    ▼
┌──────────────┐
│  生成报告     │  行为摘要 + 风险评估
└──────────────┘
```

#### 3.9 验收标准

- [ ] 沙箱环境完全隔离，不影响主机
- [ ] 单次沙箱执行 < 60 秒
- [ ] 行为记录完整（文件、网络、进程）
- [ ] 支持 Docker 和非 Docker 两种模式

## 4. 商业模式参考

| 模式 | 说明 | 参考 |
|---|---|---|
| **Open Core** | Core 开源，Pro/Enterprise 收费 | GitLab, Elasticsearch |
| **SaaS 托管** | 提供云端 SkillSecurity Server | Snyk, Datadog |
| **支持与咨询** | 免费使用，付费获取支持 | Red Hat |
| **混合** | Open Core + SaaS + 支持 | 推荐 |

**定价参考**（如选择收费模式）：

| 层级 | 价格 | 包含 |
|---|---|---|
| Core | 免费 | 全部开源功能 |
| Pro | $49/月/团队 | Web 界面 + 性能优化 |
| Enterprise | 联系销售 | 全部功能 + SLA + 定制 |
